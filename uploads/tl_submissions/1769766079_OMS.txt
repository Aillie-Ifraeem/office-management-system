# 🔑 Core Idea (very important)

All roles use **the SAME system**
What changes is:

* what they can **see**
* what they can **do**

So we don’t build 5 different apps — we build **one system with role-based screens**.

---

# 👤 Roles in Your OMF (Clean & Realistic)

We’ll define **5 roles**:

1. Admin
2. Manager
3. Team Leader
4. Employee
5. Finance

Team Leader = Employee + extra powers
Finance = Read-only + calculations

---

## 1️⃣ Admin (System Owner)

### What Admin REALLY does (no daily office work)

Admin is not HR or Manager.

Admin handles:

* System setup
* Users
* Roles

### Admin can:

* Create users
* Assign roles (Employee / TL / Manager / Finance)
* Activate / deactivate accounts
* View overall stats (counts only)

❌ Admin does NOT:

* Assign tasks
* Review work
* Chat about work

This keeps Admin **simple & clean**.

---

## 2️⃣ Employee (Base Role)

You already defined this well.
This is the **foundation role**.

Employee can:

* Mark attendance
* View assigned tasks
* Submit work
* Send messages
* Share solutions
* Write applications

Every other role **inherits** this.

---

## 3️⃣ Team Leader (Employee + Control)

Think of Team Leader as:

> “Senior employee who coordinates work”

### Team Leader can do EVERYTHING an Employee can, PLUS:

* Assign tasks to team members
* Review employee submissions
* Forward approved work to Manager
* Comment on submissions
* View team attendance
* Respond to applications

This is **very realistic**.

---

## 4️⃣ Manager (Decision Maker)

Manager is NOT coding, chatting all day.

Manager:

* Reviews summaries
* Approves/rejects
* Makes decisions

### Manager can:

* View department attendance summary
* Assign tasks to TLs
* Review forwarded work
* Approve / reject submissions
* Approve / reject applications
* Send announcements

Manager does NOT:

* Chat casually
* Submit work

Keeps role **professional**.

---

## 5️⃣ Finance Department (Data Viewer)

Finance is:

> “Logic + data, no interaction”

### Finance can:

* View attendance records
* Calculate payable days
* View approved leaves
* Generate monthly summaries

Finance does NOT:

* Chat
* Assign tasks
* Modify work

This makes Finance **simple & powerful**.

---

# 🧠 Key Design Insight (THIS IS GOLD)

Instead of:

> “Different systems for different roles”

We do:

> **One system + permission-based screens**

Example:

* If role = Employee → show Employee Dashboard
* If role = Team Leader → show extra buttons
* If role = Manager → show approval panels
* If role = Finance → show reports

This is **exactly how real apps work**.
